CS 210 Portfolio Project: Corner Grocer

Project Summary

For my CS 210 portfolio, I chose my Corner Grocer project. This program was made to help a grocery store keep track of how often each item was purchased. The program reads item names from a text file and then gives the user a menu with different choices. The user can search for one item, see all item counts, view a histogram, or exit the program. This project helps organize grocery purchase information in a simple and useful way.

What I Did Well

One thing I think I did well was creating a menu that is easy for the user to follow. The options are clear, and the program gives the user different ways to view the item information. I also used a class to organize the grocery item data. This helped keep the code cleaner instead of putting everything in one large section.

Where I Could Improve

One area I could improve is input validation. For example, if the user types a letter instead of a number, the program could handle that mistake better. I could also make the file name easier to change if the program needed to use a different input file later. These improvements would make the program easier to use, safer, and easier to update in the future.

Challenges and How I Overcame Them

The most challenging part of this project was reading the file and making sure each item was counted correctly. Using a map with pointers was also difficult because it was new for me. I worked through this by testing the program step by step and fixing one problem at a time. I also used zyBooks, class examples, instructor announcements, and practice to help me understand what I needed to do.

Transferable Skills

This project helped me build skills that I can use in future courses, projects and my career. I practiced file input and output, classes, functions, maps, loops, and menus. These skills are useful because many programs need to read data, store it, and display results in a way that makes sense to the user.

Maintainability, Readability, and Adaptability

I made the program easier to read by using clear names for variables and functions. I also used a class to separate the grocery item information from the rest of the program. This makes the code easier to understand and update. The menu design also makes the program adaptable because more options could be added later if needed.
